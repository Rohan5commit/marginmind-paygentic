# MarginMind Business Plan (Hackathon Scope)

## 1. Product Thesis
Startups and solo operators leak cash across recurring SaaS, cloud usage drift, duplicate tools, and missed refund windows. MarginMind converts spend data into direct money-saving actions with a Locus-native execution layer.

## 2. Core Value Proposition
- Detect avoidable spend quickly from CSV or connected billing exports.
- Prioritize actions by expected savings and confidence.
- Let an agent fund low-cost research/escalation work to unlock larger savings.
- Keep human approval in the loop for expensive or irreversible actions.

## 3. Economic Model
- Input cost: low per-run inference + optional low-cost wrapped API calls.
- Output value: reduced monthly burn and one-time recoveries.
- ROI target in MVP: each paid action proposal should project at least 4x return.

## 4. Target Customer
- Startup founders
- Indie hackers
- Creator businesses
- AI teams with many tool subscriptions and API bills
- Remote-first teams without dedicated finance ops

## 5. MVP Go-To-Market
- Launch as a self-serve web tool with a free sample-data mode.
- Add integrations (Stripe, Mercury exports, AWS billing) after hackathon.
- Position as “FinOps for small teams” with immediate savings proof in first session.

## 6. Locus Advantage
Locus makes machine-economy behavior practical:
- wallet-governed spend for agent actions
- paid wrapped APIs without vendor key sprawl
- task escalation for high-ROI human follow-through
- app discovery for Build/Hire pathways

## 7. Expansion After Hackathon
- One-click refund outreach workflows
- Automated renewal guard with live Build app execution
- Policy engine for autonomous approvals by confidence and ticket size
- Team collaboration and weekly savings digest
